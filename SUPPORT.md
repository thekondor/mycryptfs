# Supporting MyCryptFS

MyCryptFS was built to solve my own need for a native macOS client for gocryptfs. I'm sharing it publicly because I'm glad if it's useful to someone else facing the same problem.

If you find MyCryptFS valuable, here are meaningful ways you can support the project and help others benefit from encrypted filesystems.

## Spread the Word

### Share MyCryptFS

- Tell friends or colleagues who need encrypted filesystem support on macOS
- Spread the word or star the repository on Github
- Mention it in relevant forums or communities
- Share your experience with the software

### Educate About Encryption

One of the most impactful ways to support this project is to help others understand the importance of protecting their sensitive data:

**Encrypted File Containers**
- Educate people about the need to use encrypted file containers for sensitive data
- Introduce them to proven solutions like gocryptfs or VeraCrypt
- Explain how encrypted containers protect data even if the device is lost or stolen
- Share best practices for managing encryption passwords and master keys

**Full Disk Encryption**
- Encourage others to enable full disk encryption on their devices
- For macOS users: Recommend enabling FileVault in System Settings
- For Windows users: Suggest BitLocker or VeraCrypt
- For Linux users: Point them to LUKS or other dm-crypt solutions

**Why This Matters**
Many people don't realize how vulnerable their data is until it's too late. By educating others about encryption, you're not just supporting MyCryptFS - you're helping protect people's privacy, financial information, personal documents, and business data.

Even if someone doesn't need gocryptfs specifically, helping them enable FileVault or similar full-disk encryption is a valuable contribution to digital security. Ensure the password or the recovery key is stored safely :).
